Keyword test page for seo

This page enumerates tricky cases for matching the keyword seo. Use the attributes to drive automated tests:

  • data-scope: the assumed matcher configuration for this block (e.g., exact, fuzzy, allow-compounds).
  • data-expected on each line: whether the instance should be matched by a well-behaved matcher under the given scope.

Exact-word mode (strict)

Match only the standalone word seo with standard word boundaries. Do not match possessives/plurals or compounds.

  • Sentence with punctuation boundaries: “We do seo.”
  • Possessive (curly apostrophe): seo’s
  • Straight apostrophe: seo's
  • Leading apostrophe (elision): seo’
  • Plural: seos
  • Hyphenated prefix: seo-friendly
  • Hyphenated suffix: friendly-seo
  • Compound/keyword-like: seoish, seo‑adjacent
  • Embedded in another token: notseo, seaon
  • Alphanumeric: seo123, xseo
  • Plain word: seo

Flexible mode (inflections allowed)

Treat possessives and simple inflections as matches; still avoid compounds by default.

  • Plain word: seo
  • Possessive (curly apostrophe): seo’s
  • Straight apostrophe: seo's
  • Leading apostrophe: seo’
  • Plural: seos
  • Hyphenated prefix: seo-friendly
  • Hyphenated suffix: friendly-seo
  • Compound/keyword-like: seoish, seo‑adjacent

Flexible mode with hyphenated compounds enabled

When explicitly configured, hyphenated forms should match, but glued compounds and unrelated embeddings should still not.

  • Hyphenated prefix: seo-friendly
  • Hyphenated suffix: friendly-seo
  • Glued compound (no hyphen): seoish
  • Embedded in larger token: notseo

Notes

  • Use Unicode-aware boundary detection so seo is recognized next to punctuation and across locales.
  • Handle both straight (') and curly (’) apostrophes for possessives/contractions.
  • Hyphen handling should be configurable; many matchers treat hyphen as a non-word boundary by default—be explicit.
